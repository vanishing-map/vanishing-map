# vanishing_map 실행 계획

## 팀
- 사회혁신가: 이젠
- 개발자: 고경민

## 문제
- 실제 업무 문제: 한국의 귀농·귀촌 및 관계인구 정책에서 전국 단위 출발지→도착지 이동 흐름을 한눈에 볼 수 있는 통합 시각화가 부족해, 각 지자체가 자체 데이터만으로 유입·유출을 해석하고 과잉 경쟁하는 문제가 있다.
- 가장 어려움을 겪는 사람: 전국 이동 흐름을 바탕으로 지역 정책과 홍보 방향을 판단해야 하는 청도군 실무자
- 문제가 발생하는 순간: 특정 보고 순간이 아니라, 실무자가 언제든 전국 단위 인구 이동 유량을 bird-eye view로 보고 사람들이 어디에서 어디로 흐르는지 직관적으로 파악해야 할 때

## 자료 확인
- 자료 상태: 공공데이터 예정 및 공개 통계 샘플
- 샘플 입력: KOSIS에서 사용자가 직접 수집한 전국 출발지→도착지 이동량 CSV와 공공데이터포털 행정안전부 지역별 인구이동 현황 API 샘플. MVP에서는 기존 data/sample-2025-flows.csv 또는 같은 구조의 표준 flow CSV를 사용한다. 기본 필드는 origin, destination, people이며, 확장 필드는 year/month, characteristic, age_group, movement_reason, data_source, confidence, source_status처럼 별도 컬럼으로 둔다.
- 개인정보 처리: 공공 통계와 집계 이동량만 사용하므로 개인정보 처리 해당 없음
- 자료 유효성 확인: 민감정보가 없는 집계 데이터이며, 3시간 MVP에서는 origin, destination, people을 필수 컬럼으로 표준화한 샘플만 사용한다. 색상으로 구분할 특성 값은 people 안에 섞지 않고 characteristic 같은 별도 컬럼으로 분리한다. 공식 통계 주장에는 KOSIS 원자료로 교체 후 검증이 필요하다.

## 현재 업무 흐름
- 입력 자료: KOSIS 인구이동 통계, 공공데이터포털 행정안전부 지역별 인구이동 현황 API, 지자체별 자체 귀농·귀촌 자료
- 처리 순서: 각 기관 데이터 확인 -> 지역별 유입·유출 수치 비교 -> 특정 지역 중심으로 해석 -> 정책·홍보 방향 검토
- 판단 기준: 어느 출발지에서 어느 도착지로 이동량이 큰지, 특정 지역이 전국 흐름에서 어떤 위치인지, 강한 유입·유출 허브가 어디인지
- 현재 산출물: 지역별 유입·유출 해석, 정책 검토용 설명, 홍보 타깃 지역 가설
- 현재 사용 도구: KOSIS, 공공데이터포털, 스프레드시트, 수동 분석

## 결과물 형태
- 문제 유형: 데이터 정리/분석/대시보드
- 선택한 결과물 형태: 대시보드
- 선택 이유: 문제의 핵심이 개별 지역 조회가 아니라 전국 단위 이동 유량을 한눈에 해석하는 것이므로, 표나 필터 중심 도구보다 지도 기반 flow map 대시보드가 3시간 데모와 현장 문제에 맞다.

## 기대 변화
- 기대 변화: 전국 단위 인구 이동 흐름을 표가 아니라 대한민국 지도 베이스맵 위에, 3D 지구본 없이, 살짝 떠 있는 3D 느낌의 곡선 arc로 보여줘 정책 담당자가 큰 유량과 방향을 즉시 파악한다.
- 관찰 가능한 개선: 샘플 데이터를 넣었을 때 남한 베이스맵 위에 주요 출발지→도착지 흐름이 위로 솟았다 내려오는 곡선 arc로 나타나고, 유량 굵기와 강한 유입·유출 허브가 시각적으로 드러난다.

## 3시간 MVP
- 해결할 좁은 문제: 공개 통계 샘플 1개를 표준 flow 데이터로 넣어 전국 단위 출발지→도착지 이동 유량을 3D 지구본 없이 대한민국 지도 베이스맵 위의 곡선 arc 대시보드로 보여준다.
- 핵심 기능 1개: 한국(남한) 베이스맵 위에 출발지→도착지 이동을 위로 솟았다 내려오는 3D 느낌의 곡선 arc로 그리고, 선 굵기는 유량으로, 색상은 향후 특성 레이어로 분리해 표시하는 bird-eye flow map
- 포함할 업무 단계: 표준 flow CSV 로드 -> 남한 베이스맵과 지역 좌표 매핑 -> 유량 기준 arc 굵기 계산 -> 출발지에서 도착지로 향하는 상승-하강 곡선 arc 렌더링 -> 특성 컬럼 기반 색상/범례 설정 -> 주요 허브 요약 표시
- 데모 입력: origin, destination, people 필드를 가진 전국 이동량 샘플 CSV. 가능하면 KOSIS 직접 수집 데이터, 없으면 data/sample-2025-flows.csv 구조의 데모 데이터. 이후 특성 구분은 characteristic, age_group, movement_reason, data_source 같은 선택 컬럼으로 확장한다.
- 데모 산출물: 한국(남한) 베이스맵 위 이동 arc, 유량 굵기, 주요 유입·유출 허브, 강한 흐름 요약, 색상 범례가 확장 가능한 대시보드 화면
- AI 역할: 데이터 구조 표준화, 흐름 요약 문구 생성, 시각화 구현 보조
- 사람 검토 지점: 청도군 실무자 또는 정책 담당자가 지도에서 큰 흐름과 허브 해석이 실제 데이터 의미와 맞는지 최종 확인한다.
- 성공 기준: 전국 이동 데이터 샘플을 넣었을 때 한국(남한) 베이스맵 위에 출발지→도착지 arc가 유량에 따라 굵기로 표시되고, arc가 위로 솟았다 내려오는 3D 느낌을 주며, 색상은 향후 특성 구분을 위한 범례 슬롯으로 분리되고, 주요 이동 허브와 강한 흐름이 한눈에 보여야 성공이다.
- 실패 신호: 3D 지구본처럼 보이거나, 단순 표나 필터 화면만 나오거나, 전국 흐름의 방향과 규모가 직관적으로 보이지 않거나, arc가 평면 직선이라 이동 유량의 bird-eye 느낌이 약하거나, 색상이 유량과 특성을 뒤섞어 해석을 혼란스럽게 만들거나, 데모 데이터가 실제 통계처럼 오해되면 실패로 본다.
- 3시간 안/밖 판단: 축소 필요 - 남한 베이스맵 위 arc flow map 자체는 가능하지만 3D 지구본, 실시간 수집, KOSIS 자동화, 관계인구 추세, 정교한 행정구역 지도까지 포함하면 3시간 밖이다. 샘플 CSV 기반 정적 대시보드로 자르면 3시간 안이다.

## 제외 범위
- 오늘 하지 않을 것: 3D 지구본, 실시간 데이터 수집, KOSIS 자동 크롤링, 공공데이터 API 운영 연동, 관계인구 분기별 추세 분석, 정책 추천 자동화, 로그인/DB/배포, 모든 시군구 좌표 정밀 보정
- 다음 단계 아이디어: KOSIS 원자료 업로드 변환기, 청도군 중심 drill-down, 관계인구 체류 데이터와 인구이동 흐름 비교, 공식 통계 테이블 교체

## 3시간 구현 계획
- 0-30분: 데모 데이터 구조를 origin, destination, people 필수 컬럼과 characteristic 등 선택 컬럼으로 고정하고 기존 샘플 CSV와 KOSIS 수집 CSV 중 오늘 사용할 입력을 확정한다.
- 30-90분: 한국(남한) 베이스맵 또는 남한 윤곽 좌표 캔버스에 지역 노드와 출발지→도착지 arc를 그린다. arc는 중간 제어점을 위로 올린 곡선으로 만들어 3D처럼 솟았다 내려오는 느낌을 주고, 선 굵기는 people, 색상은 characteristic 같은 특성 컬럼에만 배정한다.
- 90-150분: 주요 유입·유출 허브, 상위 흐름 요약, 색상 범례 슬롯을 추가하고, 데모 데이터가 demo/public source인지 명확히 표시한다. 지구본이 아니라 남한 지도 위 arc flow로 보이는지 조정한다.
- 150-180분: 대시보드 화면을 검수한다. 한국(남한)만 보이는지, arc가 위로 솟았다 내려오는 3D 느낌을 주는지, 선 굵기와 색상 의미가 분리되어 보이는지, 단순 필터 화면처럼 보이지 않는지 확인하고 발표용 설명을 정리한다.

## 구현자 지침
- 구현 시작 전 이 `PLAN.md`와 `workshop.json` 초안을 먼저 읽습니다.
- 새 세션에서 구현하더라도 구현 중 결정, 변경, 막힌 점, 검증 결과를 `MEMORY.md`에 append-only로 기록합니다.
- `MEMORY.md`가 없으면 새로 만들고, 있으면 기존 내용을 덮어쓰지 않고 파일 끝에만 추가합니다.
- 구현이 끝나면 `workshop.json`의 구현 결과 관련 필드를 갱신할 수 있게 변경 요약과 검증 결과를 남깁니다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: KOSIS 원자료 변환기 추가
- 후보 1 시작 조건: flow map이 샘플 CSV로 정상 표시되고 40분 이상 남았을 때
- 후보 1 구현 내용: KOSIS에서 내려받은 CSV 컬럼을 origin, destination, people 필수 구조와 characteristic 등 선택 특성 컬럼으로 변환하는 업로드/파싱 기능을 만든다.
- 후보 1 데모 산출물: 사용자가 수집한 KOSIS CSV를 넣으면 대시보드용 flow CSV로 변환되는 화면 또는 결과 파일
- 후보 1 예상 추가 시간: 40분
- 후보 1 검수 기준: 최소 1개 KOSIS 샘플 파일이 표준 flow 구조로 변환되고, 유량과 특성 컬럼이 분리된 상태로 지도에 반영된다.
- 후보 2 작업: 청도군 중심 강조 모드 추가
- 후보 2 시작 조건: 대한민국 지도 베이스맵 위 arc flow map이 안정적으로 보이고 30분 이상 남았을 때
- 후보 2 구현 내용: 청도군과 연결된 유입·유출 흐름만 강조하고 나머지는 흐리게 보여주는 보기 모드를 추가한다.
- 후보 2 데모 산출물: 전국 흐름 속에서 청도군과 연결된 주요 출발지/도착지가 강조된 지도
- 후보 2 예상 추가 시간: 30분
- 후보 2 검수 기준: 청도군의 주요 유입·유출 arc가 남한 베이스맵 위에서 즉시 구분된다.
